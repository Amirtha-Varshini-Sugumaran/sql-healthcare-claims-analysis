**Medical Insurance Claims Analysis (SQL Project)**
Project Summary:
This SQL project analyzes a dummy dataset of 30 medical insurance claims to extract KPIs, identify anomalies, evaluate approval rates, and understand claim patterns across members, claim types, and statuses.

Business Questions Answered
1)Total number of claims processed
2)Claim distribution by status (approved, denied, pending, under review)
3)Average claim amount per claim type
4)Highest claim amount + detailed record
5)Daily submitted claim trends
6)Total claim value per member
7)High-value pending claims (> ₹10,000)
8)Anomalous claims (> ₹15,000)
9)Approval rate per claim type
10)Ranking of claim types by total claim cost

Domain Knowledge: Healthcare / Insurance

This analysis reflects common KPIs in insurance analytics:
Claim approval ratios
Fraud detection indicators
High-value claim alerts
Claim type cost distribution
Member risk pattern analysis
Daily throughput trends

Tech Stack:
SQL (MySQL)
Data Cleaning & Aggregation
KPI Measurement
Conditional Aggregates
Subqueries

Insights (Summary of Findings)
Inpatient & Surgical claims show highest amounts
A few claims exceed ₹15,000 → potential fraud/anomaly indicators
"Under Review" category holds a significant portion → process efficiency issue
Member-level spending varies widely → possible risk segmentation use-case
Optical and Dental claims form a major percentage of total claim count
